# RAG (Retrieval-Augmented Generation)

## RAG의 개념

RAG: 내가 데이터를 줄테니, 마치 이 정보를 아는 것처럼, 답변을 생성해줘.

### Retrieval (검색)

- 데이터를 가져오는 것
- 언어모델이 답변/아웃풋을 만드는데 필요한 정보를 제공하는 것
- 언어모델이 답변 생성에 능숙하지만, 답변 생성을 위한 모든 정보를 가지고 있지 않다

### Augmented (증강)

- 검색된 정보를 LLM의 입력에 추가하여 컨텍스트를 풍부하게 만드는 것
- 모델이 더 정확한 정보를 가지고 답변할 수 있도록 함

### Generation (생성)

- LLM이 생성하는 것

## RAG가 필요한 이유

- **Hallucination 문제 해결**: 정확한 정보를 미리 제공하여 거짓 정보 생성 방지
- **최신 정보 반영**: 학습 데이터에 없는 최신 정보도 활용 가능
- **신뢰도 향상**: 출처가 명확한 정보 기반의 답변

## 우리가 해야할 일

답변은 LLM이 생성하는 것이므로, **우리가 해야할 일**:

1. 데이터를 **잘 가져오기**
2. 데이터를 **잘 전달하기**

### 잘하려면?

- 일단 **잘 저장해야 함** (데이터 저장)
- **프롬프트 잘 활용해야 함**
- **문맥을 어떻게 제공할 것인가** (Context 관리)
