# 벡터 데이터베이스와 유사도 검색

## 답변 생성에 필요한 데이터

- **사용자가 원하는 정보** = 사용자의 질문과 관련있는 데이터
- 챗봇을 만들기 때문에, 질문과 관련된 데이터를 찾는 것이 중요

## 관련 데이터 판별 방법

벡터를 활용해서 **유사도(Similarity)**를 파악하자.
참고: [TensorFlow Projector](http://projector.tensorflow.org/)

## 벡터 생성

### 임베딩 모델 (Embedding Model)

- 임베딩 모델을 활용해서 **벡터 생성**
- 텍스트를 고차원 공간의 벡터로 변환

## 벡터 데이터베이스 (Vector Database)

### 역할

- 임베딩 모델을 활용해 생성된 벡터를 저장

### 중요: 메타데이터와 함께 저장

- 벡터만 저장하면 안됨
- 문서의 이름, 페이지 번호 등의 메타데이터를 **함께 저장**
- **왜?** LLM이 답변을 생성할 때 근거(출처)를 함께 제시할 수 있어서 답변의 퀄리티 향상

## 유사도 검색 (Similarity Search)

### 개념

- 사용자의 질문과 유사한 **문서를 가져오는 것**

### 문서 청킹의 중요성

- 전체 문서를 활용하는 것은 **속도가 느림**
- 토큰 수 초과로 **답변 생성이 안될 수도 있음**
- **문서 청킹**: 문서를 작은 단위로 나눠서 저장하는 것이 중요

## 내가 만들 RAG 시스템 ✅

### 1. Retrieval (검색)

- 사용자의 질문과 가장 비슷한 문서를 가져오는 것

### 2. Augmented (증강)

- 가져온 문서를 prompt를 통해 LLM에 제공

### 3. Generation (생성)

- LLM은 프롬프트를 활용해 답변 생성
