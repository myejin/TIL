> Front-End Development

- HTML, CSS, JavaScript 를 활용해서 데이터를 볼 수 있게 만들어 줌
  - 이 작업을 통해 사용자는 데이터를 눈으로 볼수 있고, 상호작용할 수 있음



> Vue.js

- SPA(Single Page Application)를 완벽하게 지원
- [참고] Evan You, 구글의 Angular 개발자 출신



>SPA (Single Page Application) 단일 페이지 어플리케이션

- 현재 페이지를 동적으로 렌더링함으로써 사용자와 소통하는 웹 어플리케이션
- 단일 페이지로 구성되며, 서버로부터 최초에만 페이지를 다운로드하고, 이후에는 <b>동적으로 DOM 구성</b>
- 연속된 페이지 간 사용자경험 향상
  - 트래픽 감소, 속도, 사용성, 반응성의 향상
- 동작 원리의 일부가 CSR(Client Side Rendering)의 구조를 따름



> SPA 등장 배경

- 과거 웹 사이트들은 요청에 따라 매번 새로운 페이지를 응답하는 방식
  - MPA (Multi Page Application)
- 스마트폰 등장 -> 모바일 최적화 필요 -> `모바일 네이티브 앱` 형태의 웹페이지가 필요해짐



> CSR

- 서버에서 화면을 구성하는 `SSR` 방식과 달리 클라이언트에서 화면을 구성
- 최초 요청 시 HTML, CSS, JS 등 데이터를 제외한 각종 리소스를 응답받고, 이후 클라이언트에서는 필요한 데이터만 요청해 `JS로 DOM을 렌더링`하는 방식
  - `브라우저에서 동적으로 DOM을 그린다.`
- SPA가 사용하는 렌더링 방식
-  동작방식
  - 서버가 브라우저로 Response 보낸다.
  - 브라우저가 JS를 다운로드한다.
  - 브라우저가 React를 실행시킨다.
  - 페이지가 `Viewable`, `Interactable` 한다.
- 장점
  - 서버와 클라이언트 간 트래픽 감소
    - 정적 리소스는 최초에 한번 다운로드 후 필요한 데이터만 갱신
  - 사용자 경험(UX) 향상
    - 전체 페이지를 다시 렌더링하지 않고 변경 부분만 갱신
- 단점
  - SSR에 비해 전체 페이지 렌더링 시점이 느림
  - `SEO(검색 엔진 최적화)`에 어려움이 있음
    - 최초 문서에 데이터가 없기 때문



> SSR

- Server Side Rendering
- JS 웹 프레임워크 이전에 사용되던 전통적인 렌더링 방식
- 장점
  - 초기 구동속도가 빠르다.
  - SEO(검색 엔진 최적화)에 적합
    - DOM에 이미 모든 데이터가 작성되어있기 때문
- 단점
  - 모든 요청마다 새로운 페이지를 구성해서 전달
    - 반복되는 전체 새로고침 -> UX 저하
    - 트래픽이 많아져 서버의 부담이 커진다.



> SSR & CSR

- Django에서 Axios를 활용한 좋아요/팔로우 로직의 경우,
  - 대부분 서버에서 완성된 HTML을 제공하는 구조 (`SSR`)
  - 단, 특정 요소(좋아요/팔로우)만 JS(AJAX & DOM 조작)을 활용 (`CSR`)
    - AJAX를 활용해 비동기 요청으로 필요한 데이터를 클라이언트에서 서버로 직접 요청을 보내 받아오고 JS를 활용해 DOM을 조작



> [참고] SEO

- Search Engine Optimization (검색 엔진 최적화)
- 웹 페이지 검색엔진이 자료를 수집하고 순위를 매기는 방식에 맞게 웹페이지를 구성해서 `검색결과의 상위에 노출`될 수 있도록 하는 작업
- 인터넷 마케팅 방법 중 하나
- 구글의 등장 이후 검색엔진들이 `컨텐츠의 신뢰도를 파악`하는 기초 지표로 사용됨
  - 다른 웹 사이트에서 `얼마나 인용되었나`를 반영
  - 타 사이트에서 인용되는 횟수를 늘리는 방향으로 최적화



> [참고] SEO 대응

- Vue.js 등 SPA 프레임워크는 SSR을 지원하는 SEO 대응 기술이 이미 존재
  - SEO 대응이 필요하다면 그 페이지에 대해 선별적 SEO 대응 가능
- 추가로 별도의 프레임워크를 사용하기도 함
  - Nuxt.js
    - Vue.js 응용 프로그램을 만들기 위한 프레임워크
    - SSR 지원
  - Next.js
    - React 응용 프로그램을 만들기 위한 프레임워크
    - SSR 지원



> 프론트엔드 웹 프레임워크, 왜 사용할까?

- 예시) 페이스북 친구가 이름을 수정했다면
  - 타임라인의 이름, 페이스북 메세지 상 이름, 내 주소록에서의 친구 이름 등 변경
  - 하나의 변경으로 인해 상호작용돼서 변경되는 데이터들이 너무 많다.
    - 페이스북이 React를 개발한 이유



> Vanilla JS와 비교

- Vanilla JS
  - 한 유저가 100만개의 게시글을 작성했다면
    - 이 유저가 닉네임을 변경하면, 게시글 100만개 모두 수정되어야 한다.
    - '모든 요소'를 선택해서 '이벤트'를 등록하고 값을 변경해야함
- Vue.js
  - `DOM과 데이터가 연결되어 있다면`
  - 데이터를 변경하면 이에 연결된 DOM은 알아서 변경
  - 즉, 우리가 신경 써야 할 것은 오직 데이터에 대한 관리

> MVVM Pattern

- 어플리케이션 로직을 UI로부터 분리하기 위해 설계된 디자인 패턴
- 구성 요소
  - Model : plain JavaScript Object (키-벨류 형태)
  - View : DOM (HTML)
    - 데이터의 변화에 따라서 바뀌는 대상
  - View Model : Vue (DOM과 Data의 중개자)
    - 뷰모델은 모든 Vue 인스턴스다.
    - 데이터를 얼마만큼 잘 처리해서 보여줄 것인지(DOM)를 고민하는 것



> Vue 인스턴스

- 모든 Vue 앱은 Vue함수로 새 인스턴스를 만드는 것부터 시작

  ```javascript
  const app = new Vue({
    //
  })
  ```

- Options 객체를 전달한다. `{}`
- 여러 Options들을 사용하여 원하는 동작 구현
- `Vue instance = Vue Component` 



> Options/DOM - 'el'

- Vue 인스턴스에 연결(마운트)할 기존 DOM 요소가 필요
- CSS 선택자 문자열 또는 HTML 요소로 작성
- new를 이용한 인스턴스 생성 때만 사용



> Options/Data - 'data', 'methods'

- `data` : Vue 인스턴스의 데이터 객체 (Model)
- `methods` : Vue 인스턴스에 추가할 메서드
- Vue template 에서 interpolation(보간법, 예 - `{{ }}`) 을 통해 접근 가능

- Vue 객체 내 다른 함수에서 `this` 키워드를 통해 접근 가능
- 주의
  - 화살표 함수를 `data`와 `method 정의`에서 사용하면 안된다.
  - 화살표 함수가 Vue 인스턴스의 부모 컨텍스트를 바인딩하기 때문에
    - `this`가 Vue 인스턴스를 가리키지 않음



> 'this'

- Vue 함수 객체 내에서 vue 인스턴스를 가리킴
- [주의] JavaScript 함수에서의 this 키워드는 다른 언어와 조금 다르게 동작
  - [참고] `addEventListener` 의 콜백함수에서도 `function 키워드`를 사용하자.



> Options/Data - 'computed'

- 데이터를 기반으로 하는 계산된 속성
- methods 와의 차이점 : 종속된 데이터에 따라 `저장(캐싱)`된다.
  - `종속된 데이터가 변경될 때만 함수를 실행한다.`
- 반드시 반환 값이 있어야 한다.



> Template Syntax

- 렌더링된 DOM을 기본 Vue 인스턴스의 데이터에 선언적으로 바인딩할 수 있는 HTML 기반 템플릿 구문 사용
  - `Interpolation`, `Directive` 



> 선언형 & 명령형

- 선언형 : 계산해야 하는 목표 데이터 정의, `computed`
- 명령형 : 데이터가 바뀌면 특정 함수 실행, `watch`

> Lifecycle Hooks

- 각 Vue 인스턴스는 생성될 떄 일련의 초기화 단계를 거침
  - 데이터 관찰 설정이 필요한 경우
  - 인스턴스를 DOM에 마운트하는 경우
  - 데이터가 변경되어 DOM을 업데이트 하는 경우 등
- 그 과정에서 사용자 정의 로직을 실행할 수 있는 Lifecycle Hooks 도 호출된다.
